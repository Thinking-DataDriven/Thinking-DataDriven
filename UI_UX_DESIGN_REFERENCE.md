# UI_UX_DESIGN_REFERENCE.md

## Назначение файла
Этот файл — обязательный дизайн-референс для проекта Thinking OS / MindOS.

Используй его как главный источник истины по UI/UX.
Не переизобретай стиль.
Не меняй структуру экранов без необходимости.
Задача — воспроизвести интерфейс максимально близко к согласованному preview-концепту.

---

# 1. Общая дизайн-идея

Нужен интерфейс в стиле:

- premium SaaS
- clean
- thoughtful
- calm
- desktop-first
- light interface
- много воздуха
- мягкие тени
- крупные скругления
- спокойная визуальная система
- ощущение дорогого продукта для мышления, а не обычного таск-менеджера

Ключевое ощущение:
**спокойный, умный, собранный, премиальный интерфейс для глубокого мышления.**

Нельзя делать:
- слишком яркий startup-style интерфейс
- кричащие цвета
- перегруженный dashboard
- слишком много иконок
- плотную сетку без воздуха
- визуал “обычного CRM”
- визуал “chat app”
- кислотные градиенты
- дешевые glassmorphism-эффекты

---

# 2. Базовая визуальная система

## Фон страницы
Использовать светлый мягкий фон с легким градиентом:

- from-slate-100
- via-slate-50
- to-white

То есть общий фон:
очень светлый, мягкий, с легким глубинным переходом.

## Главная палитра
Основной набор:
- slate
- white
- neutral light gray
- very soft accent only where needed

Темная навигация:
- sidebar: slate-950 / почти черный
- текст на sidebar: white / slate-300 / slate-400

Карточки:
- белые
- border-slate-200
- very soft shadow

Текст:
- заголовки: slate-900
- вторичный текст: slate-500 / slate-600

Акценты:
- использовать редко
- только для статуса, фокуса недели, AI-note, recommended skill

## Скругления
Очень важны большие скругления:
- большие контейнеры: rounded-3xl
- sidebar: rounded-[28px]
- pills: rounded-full
- кнопки: rounded-xl / rounded-2xl

## Тени
Легкие и дорогие:
- shadow-sm на карточках
- shadow-2xl можно только на sidebar или крупных блоках

# 3. Layout приложения

Нужен desktop-first layout.

## Структура
Главный экран должен состоять из:
- верхнего хедера
- левого sidebar
- основной контентной области

На desktop:
- sidebar слева фиксированной ширины примерно 272–288 px
- справа основной контент
- большие внешние отступы
- max width большой, примерно до 1600 px

Пример логики layout:
- page padding: около 24 px
- main container: centered
- grid:
  - left column = sidebar
  - right column = main content

## Sidebar
Sidebar — очень важный визуальный якорь.

Он должен быть:
- темный
- высокий
- premium
- с внутренними отступами
- sticky сверху
- с четкими nav buttons
- с карточкой текущего bottleneck
- с AI coach note внизу

Sidebar не должен быть:
- узким
- перегруженным иконками
- слишком технологичным
- похожим на admin panel

---

# 4. Верхний header

Сверху нужен header:
- большой заголовок страницы / продукта
- подзаголовок
- справа или под ним набор переключателей экранов

В header должно быть видно:
- название концепта / продукта
- короткое пояснение
- быстрые кнопки-переключатели экранов

Стиль header:
- очень чистый
- без перегруза
- с хорошей типографикой
- без лишних декоративных элементов

---

# 5. Типографика

Нужна чистая иерархия.

## Заголовки
- крупные
- semibold
- slate-900

Примеры:
- page title: text-3xl font-semibold
- section title: text-lg font-semibold

## Основной текст
- text-sm / base
- вторичный текст slate-500 / slate-600
- удобная межстрочность

## Микротекст
- text-xs
- slate-400 / slate-500
- для labels, meta info, source types, status captions

Интерфейс должен читаться легко.
Нельзя делать мелкий плотный текст.

---

# 6. Кнопки и pills

## Кнопки
Кнопки должны быть:
- мягкие
- rounded
- минималистичные
- с явным hover
- без агрессивных теней

Primary button:
- bg-slate-950
- text-white
- rounded-2xl
- комфортный padding

Secondary button:
- white background
- border
- text-slate-700
- hover:bg-slate-50

## Pills
Pills использовать для:
- тэгов
- приоритетов
- областей
- статусов
- skill labels

Стиль pill:
- rounded-full
- border-slate-200
- bg-slate-50
- text-xs
- font-medium

# 7. Навигация экранов

Нужно 5 основных экранов:

1. Главная
2. Думать с ИИ
3. Лента знаний
4. Карта роста
5. Обзоры и паттерны

Навигация должна работать:
- и через sidebar
- и через top buttons в header

Активный экран должен:
- явно выделяться
- иметь темный active state для верхних кнопок
- иметь белый active state внутри темного sidebar

# 8. Экран: Главная

## Смысл
Это центральный dashboard.
Он не должен быть перегруженным.
Он должен давать чувство:
“я сразу понимаю свой фокус, bottleneck и что делать дальше”.

## Структура экрана
1. Заголовок секции
2. Верхний ряд из 4 stat cards
3. Большой блок “Главная рекомендация”
4. Справа блок “Найденные паттерны”
5. Нижний ряд из 3 блоков:
   - Активные проекты
   - Последние решения
   - Выжимка из ленты

## Верхний ряд stat cards
4 карточки:
- Навык недели
- Главный bottleneck
- Качество решений
- Темп обучения

Стиль:
- белые карточки
- одинаковая высота
- компактные, но с воздухом
- label сверху маленьким текстом
- value крупно
- subtext маленьким вторичным текстом

## Главная рекомендация
Это большой hero-like блок.
Он должен визуально быть самым важным блоком экрана.

Внутри:
- маленький label “AI-синтез”
- крупный заголовок рекомендации
- абзац объяснения
- справа pills
- ниже 3 action cards:
  - Шаг 1
  - Шаг 2
  - Шаг 3

Стиль:
- outer white card
- inside slightly tinted background (slate-50)
- очень чисто
- крупно
- без перегруза

## Найденные паттерны
Вертикальный столбец карточек справа:
- Повторяющаяся ошибка
- Сильная сторона
- Слепая зона
- Возможность роста

Стиль:
- stack of small cards
- light background
- bordered
- compact but readable

## Нижний ряд
Три одинаково важные панели:
- Активные проекты
- Последние решения
- Выжимка из ленты

Все три:
- в белых карточках
- с section title
- со stack layout внутри

# 9. Экран: Думать с ИИ

## Смысл
Это не просто чат.
Это structured thinking workspace.

Нужно ощущение:
“я зашел сюда, чтобы разобраться глубже”.

## Структура
1. Заголовок секции
2. Ряд режимов:
   - Декомпозиция
   - Корневая причина
   - Разбор решения
   - Контраргумент
   - Карта системы
   - Лучший следующий шаг
3. Основной двухколоночный layout:
   - слева большая область разговора / reasoning
   - справа контекст и сохранение результатов

## Левая большая зона
Содержит:
- карточку “Ты” с темным фоном
- карточку “Thinking Partner” со светлым фоном
- внутри AI-ответа 3 explanation cards
- блок “Вопрос для тебя”
- внизу input area + primary action

Важно:
- не делать этот экран похожим на мессенджер
- сделать его похожим на thinking session board

## Правая колонка
Два блока:
1. Какой контекст использован
2. Что можно сохранить

Это должны быть компактные белые карточки со списками.

---

# 10. Экран: Лента знаний

## Смысл
Это curated feed.
Не бесконечная лента.
Не новостной шум.
А осмысленная knowledge surface.

## Структура
1. Заголовок
2. Фильтр-кнопки:
   - Все
   - AI
   - Спикеры
   - Решения
   - Системы
   - Обучение
   - Сохраненное
3. Grid карточек 3 в ряд на desktop

## Карточка knowledge item
Каждая карточка должна содержать:
- meta line: источник
- title
- summary
- pills/tags
- 4 action buttons:
  - Сохранить инсайт
  - Связать с проектом
  - Сделать принципом
  - Тренировать навык

Стиль карточек:
- белые
- rounded-3xl
- с хорошими отступами
- много воздуха
- без визуального шума

---

# 11. Экран: Карта роста

## Смысл
Показать мета-навыки и текущий фокус развития.

## Структура
2 колонки:
- слева широкий блок “Профиль мета-навыков”
- справа узкий блок “Текущий фокус”

## Профиль мета-навыков
Вертикальный список навыков с прогресс-барами.

Навыки:
- Системное мышление
- Декомпозиция
- Приоритизация
- Критическое мышление
- Рефлексия
- Second-order thinking

Каждый элемент:
- название
- score справа
- горизонтальный progress bar
- маленькое пояснение ниже

Стиль:
- спокойно
- не как gaming dashboard
- не делать слишком яркие bars
- bars темные / slate-based

## Текущий фокус
Темный акцентный блок:
- label
- название навыка крупно
- объяснение
- упражнение внутри вложенного блока

Этот блок должен быть визуально сильным, но не агрессивным.

---

# 12. Экран: Обзоры и паттерны

## Смысл
Это экран рефлексии и накопления правил.

## Структура
2 колонки:
- слева большой блок “Недельный обзор”
- справа 2 stacked blocks:
  - Повторяющиеся ошибки
  - Извлеченные принципы

## Недельный обзор
Содержит список из 5 reflection prompts:
- Что реально сдвинуло систему вперед?
- Что забрало энергию, но дало мало рычага?
- Какое решение было самым сильным и почему?
- Какая ошибка снова повторилась?
- Какой один bottleneck важнее всего на следующей неделе?

Ниже 2 кнопки:
- Запустить AI-обзор
- Сохранить ответы

## Правая колонка
### Повторяющиеся ошибки
Стек карточек со списком recurring mistakes

### Извлеченные принципы
Стек карточек с готовыми принципами

---

# 13. Поведение и интерактивность

Обязательно:
- sidebar кнопки кликабельны
- top nav buttons кликабельны
- переключение экранов должно быть реальным
- hover states должны быть мягкими
- active states должны быть явно видны

Даже если это пока mock frontend:
- структура должна ощущаться живой
- нельзя делать просто одну длинную статичную страницу

---

# 14. Адаптация под разработку

При реализации:
- использовать reusable components
- выделить layout
- выделить cards
- выделить pills
- выделить section headers
- выделить stat cards
- выделить navigation buttons

Нужна чистая структура компонентов.

Пример логики компонентов:
- AppShell
- Sidebar
- TopScreenSwitcher
- SectionTitle
- StatCard
- TagPill
- InfoCard
- RecommendationCard
- PatternCard
- KnowledgeCard
- SkillProgressItem

---

# 15. Очень важные ограничения

Codex не должен:
- менять общий стиль на темный full-dark
- делать яркий градиентный startup look
- добавлять много иконок без необходимости
- делать перегруженный интерфейс
- менять тексты экранов на английский
- превращать “Думать с ИИ” в обычный chat UI
- превращать “Карту роста” в аналитическую BI-панель
- превращать “Ленту знаний” в новостной портал

---

# 16. Главный критерий качества

Результат должен быть визуально максимально близок к следующему ощущению:

**дорогой, спокойный, умный, light premium SaaS для личного мышления и развития.**

Если при просмотре интерфейс начинает напоминать:
- CRM
- таск-менеджер
- чат-бот приложение
- админку
- crypto dashboard
- яркий startup template

значит дизайн ушел не туда.
