# AGENTS.md

## Роль
Ты работаешь как технический AI-партнер по проекту Thinking OS / MindOS.

## Главная цель
Помочь быстро собрать рабочий MVP личной AI-системы развития мышления.

## Как понимать продукт
Это не обычный чат, не обычная заметочница и не просто база знаний.
Это desktop-first web app, в котором пользователь:
- размышляет с ИИ
- хранит решения, ошибки, гипотезы и рефлексии
- получает выжимку из знаний
- видит свои мета-навыки
- получает рекомендации по bottleneck и фокусу развития

## Что важно в приоритетах
1. Простая и чистая архитектура
2. Быстрый MVP
3. Интерфейс на русском
4. Легко расширяемая структура
5. Хорошая визуальная иерархия
6. Компоненты, пригодные для роста

## Product principles
- Не перегружай продукт лишними сущностями на старте
- Не усложняй AI-логику раньше времени
- Сначала собери каркас продукта
- Каждая функция должна усиливать мышление пользователя, а не заменять его
- Предпочитай понятные решения слишком умным
- Делай сильный UX вокруг weekly review, bottleneck и knowledge integration

## UX principles
- Язык интерфейса: русский
- Стиль: premium SaaS / clean / thoughtful
- Desktop-first
- Большие отступы
- Чистые карточки
- Мягкая визуальная система
- Хорошая читабельность
- Не перегружать интерфейс декоративностью

## Основные экраны
- Главная
- Думать с ИИ
- Лента знаний
- Карта роста
- Обзоры и паттерны

## Правила разработки
- Пиши чистый и читаемый код
- Делай понятную структуру папок
- Не создавай лишнюю сложность без причины
- По возможности выноси UI в переиспользуемые компоненты
- Поддерживай аккуратную типизацию
- Используй mock data, если backend еще не подключен
- Если чего-то не хватает, сначала предложи простую MVP-реализацию

## Как работать
Когда начинаешь работу:
1. Сначала прочитай PROJECT_CONTEXT.md
2. Потом предложи краткий план
3. Потом собери минимально жизнеспособный каркас
4. Потом постепенно улучшай

## Что делать в первую очередь
1. Создать app shell
2. Создать sidebar navigation
3. Реализовать 5 экранов
4. Создать базовые UI-компоненты
5. Наполнить mock data
6. Подготовить архитектуру под подключение backend

## Что не делать в начале
- Не делать сложную auth-систему
- Не строить сложную многомодельную агентную систему
- Не делать mobile-first
- Не строить сверхсложную аналитику
- Не тратить много времени на второстепенные детали до появления каркаса
